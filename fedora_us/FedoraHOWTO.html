Fedora HOWTO

Fedora HOWTO　フェドーラを初めて使う方に

  • ご注意： fedora.us が提供するパッケージは、レッドハットおよび私達 fedora.us が提供している他のパッケージといっしょに利用することを前提として、開発され十分に検証されたものです。他のリポジトリーとの混用によって生じる互換性の問題については、Repository Mixing Problems （邦訳）をご覧になってください。

始めに、fedora.us がそれぞれのソフトウェア毎に提供している詳細について FedoraChannels というページの解説をご覧ください。

フェドーラ プロジェクトのウェブサイトからパッケージをインストールする方法としては、 Fedora tree の中からいちいちダウンロードしてインストールするという方法も選べますが、長期的な運用を考慮した場合、数種類用意されているうちのいずれかの自動更新による運用方法を習得される方がずっと簡単です。 Fedora.us では、「APT 」と「YUM 」の2種類のパッケージ管理用のツールを用意しています。どちらを使っても、指定されたパッケージのダウンロードやインストールを自動的に行えます。使用するリポジトリーは、ステーブル（安定版）にあらかじめセットされています。APT や YUM は、どちらか単独で使用することも、両方を使用することもできます。ただし、両方を同時に使用することはできません。

Current Package Managers 現行のパッケージ管理ツール

Fedora Core 1 用

  • http://download.fedora.us/fedora/fedora/1/i386/RPMS.stable/apt-0.5.15cnc1-0.fdr.3.1.i386.rpm APTを利用するためには、ウェブサイトサイトからこのパッケージをダウンロードしインストールします。

  • yum is already included in FC1. Read FedoraSources to configure fedora.us repositories. YUM の場合は、最初から Fedora Core 1 (FC1) に含まれています。fedora.us のリポジトリー設定の詳細については、 FedoraSources をご覧ください。

Red Hat Linux 9 用

  • http://download.fedora.us/fedora/redhat/9/i386/RPMS.stable/apt-0.5.5cnc6-0.fdr.8.rh90.i386.rpm APT を利用する場合、これをダウンロードしインストールします。

  • http://download.fedora.us/fedora/redhat/9/i386/RPMS.stable/yum-2.0.3-0.fdr.1.rh90.noarch.rpm YUM を利用する場合、これをダウンロードしインストールします。

Red Hat Linux 8.0 用

  • http://download.fedora.us/fedora/redhat/8.0/i386/RPMS.stable/apt-0.5.5cnc6-0.fdr.8.rh80.i386.rpm APT を利用する場合、これをダウンロードしインストールします。

  • http://download.fedora.us/patches/redhat/8.0/i386/RPMS.stable/yum-2.0.3-0.fdr.1.rh80.noarch.rpm YUM を利用する場合、これをダウンロードしインストールします。
    (requires rpm >= 4.1.1)     （ただし、インストールの際使用するRPM 管理ツールは、バージョン4.1.1 以上であることが必要です。）

お住いになっている場所の近くにある高速なミラーサイトを選ぶためには、 FedoraMirrorList というページをご覧になってください。ミラーサイトが利用できるように、設定ファイルの記述を書き換えてください。APT では、sources.list という名のファイルがディレクトリー /etc/apt/sources.list に存在します。YUM の場合の設定は、 /etc/yum.conf に記述されています。

YUM を利用する場合には、フェイルオーバー [サーバー間で異常アプリケーションを代行処理するという]機能が追加されているので、あるダウンロードサーバーが一時的に利用できない場合でも、その処理が継続されます。近くにあるミラーサーバーが最初に機能するように /etc/yum.conf のbaseurl パラメーターにそのアドレスを登録してください。

APT を利用する場合、他のプロジェクトのリポジトリーで提供されている APT を使って、すでにお使いの sources.list ファイルの記述にフェドーラのリポジトリーを追加されるよりは、フェドーラ専用に機能強化された APT パッケージをインストールされるよう、強くお勧めします。どうしてもこの点にこだわりがあるのでしたら、 FedoraSources のページをお読みください。

HOWTO use apt-get　apt-get コマンドの使い方

apt-get update
    Download package lists.  Packages change several times daily, so do this often. （利用可能な）パッケージの一覧をダウンロードします。パッケージの変更は1日の内に数回ありますので、頻繁に実行してください。
apt-get dist-upgrade
    Update all packages. インストールされている全てのパッケージを更新します。
apt-get install <packagename>
    Install/upgrade specific package(s) (and its dependencies, if missing any).
　　特定のパッケージを指定してインストールや更新を行えます。（パッケージの依存関係を含みます。）
apt-cache search <word>
    Search all known packages entries (descriptions etc) for <word>.
　　パッケージ名をもとにして、その詳細などを検索できます。
apt-cache show <packagename>
    Show basic information about a package.
　　あるパッケージについての基本情報を表示できます。

HOWTO use yum　YUMの使い方

Make sure you have yum installed and then read FedoraSources to configure.
まず、YUM がインストールされていることを確認してから、 FedoraSources をご覧になって設定してください。

Now you can run the following commands:
次のコマンドを使って操作します：

yum list
    List all available software.
　　利用可能なソフトウェアの一覧を表示させます。
yum check-update OR yum list updates
    See if there are updated packages available.
　　更新可能なパッケージがあるかどうかの確認ができます。
yum update
    Update all installed packages that have a newer version available.
　　新しいパッケージがあれば、全て更新します。
yum install <packagename>
    Install specific package (and its dependencies, if missing any).
　　指定した特定のパッケージをインストールします。（この場合、依存関係も含まれます。）

yum search <word>
    Search all known packages entries (descriptions etc) for <word>.
　　パッケージ名をもとにして、その詳細などを検索できます。
yum info <packagename>
    Show basic information about a package.
　　あるパッケージについての基本情報を表示できます。

All of the above commands can take shell-style wildcards (*,?) instead of a package name. Check the yum man page for more details.　上に挙げたコマンドを実行する際、パッケージ名の代わりに、ワイルドカードなどを使うことも可能です。詳しくは、YUM のマニュアルのページをご覧ください。

Up2date　アップ2デート

Fedora Core 1 では、YUMやAPTのリポジトリーを利用して、アップ2デートの機能が利用可能となっています。この機能を利用したい場合には、ルート権限でログインし、 /etc/sysconfig/rhn/sources の場所にあるファイルの記述に以下のサイトを追加します。大抵のユーザーは、安定用（stable）のリポジトリーしか必要としませんが、オプションとして、（検証が不十分な）不安定版のパッケージを利用できるように設定することも可能です。

yum fedora-us-stable-fc1　　　 http://download.fedora.us/fedora/fedora/1/i386/yum/stable
yum fedora-us-testing-fc1 http://download.fedora.us/fedora/fedora/1/i386/yum/testing
yum fedora-us-unstable-fc1 http://download.fedora.us/fedora/fedora/1/i386/yum/unstable
