Iceweasel ウェブブラウザで、検索入力中画面が固まる (uim-anthy) (2007.05.02) (2009.02.22 現在再現性あり)

ウェブブラウザより、検索入力中の時等で画面が固まってしまい、全く応答しなくなります。この不具合は以前から長い間指摘されて来まし た。ブラウ ザーの画面が固まるため、Ctrl + Alt + Backspace キーでアプリケーションを強制終了する以外、対処方法がありません。

入力方式を Scin + Anthy に変更すれば、この問題は発生しなくなります。次のコマンドを実行し、インストール完了後、再ログインすれば自動的に scim-anthy に切り替わります。

 # apt-get install scim-anthy kasumi

↑

X セッションからログオフできません。(Intel 845GE) (2007.04.13)

Intel 845GE ビデオチップ搭載のマザーボードで発生する可能性があります。おそらく、これは Debianインストーラー debian-40r0-i386-netinst.iso の不具合であると思われます。ビデオカードの再設定を実行すれば、解消されるようです。この変更はシステムの再起動後、有効となります。

 # dpkg-reconfigure xserver-xorg
 # reboot
  • ビデオカードの種類は、i810 を選択します。
↑

scim-anthy で入力中文字化けが発生する。 (2006.07.16) (2007.04.13 現在なお不完全)

文字入力時（とくに MediaWiki、PukiWiki の編集画面で）、画面の一部が時々文字化けすることがあります。
直前に入力した文字が、画面上に残ったままの状態で表示（実際の操作と画面表示の不一致など）されたりもします。

ただし、編集した文書をファイルを保存することは問題ありません。保存したファイルを開いても、実際に文字化けはありません。

書庫マネージャーで lzhアーカイブが展開できない。(2006.12.12) (対処可能)

書庫マネージャー（file-roller）で、lzh形式の圧縮ファイルが展開できないようです。.lzhを拡張子とするファイル は、本来書庫マ ネージャーが標準でサポートする形式です。

とりあえず、lha というファイル展開ツールを追加インストールし、再ログインしたところ、書庫マネージャーが正常に機能するようになりました。

 # apt-get install lha

Apt Watcher で更新中、突然落ちる (2006.07.15)（2006.12.12 現在修正済み）

apt-watch applet より自動更新を実行させると、突然アプレットが落ちて作業が強制終了します。

↑

Synaptic でパッケージの更新・追加ができない。 (2006.07.15)（2006.12.12 現在修正済み）

Synaptic を使ったパッケージの更新や、個々のパッケージ追加を実行しようとすると、突然落ちて作業が強制終了するようです。パッケージを削除する 場合は、問題あり ません。

インストール後、音が全く出ません。- snd-intel8x0 (2006.07.12) （2006.12.12 現在修正済み）

サウンドカードの自動設定が機能していないためです。alsa-utils をインストールして手動で設定してください。

 # apt-get install alsa-utils
 # alsaconf

　Unloading ALSA sound driver modules: snd-intel8x0 snd-ac97-codec snd-ac97-bus snd-pcm snd-timer snd-page-alloc.
　Building card database...
　
　update-modules を実行中...
　Loading driver...
　Setting default volumes...
　Saving the mixer setup used for this in /var/lib/alsa/asound.state.
　
　=================================================
　
　これで ALSA が使用可能になりました。
　音量の調整には、ミキサープログラムをご使用下さい。

音楽CDの再生が不能になります。- GNOME環境 (2006.07.12) (2007.01.26 現在修正済み)

Debian のテスト版は公式リリースではないため、仕様の一時的な変更等のため頻繁に不具合が発生します。これも、その一例です。

システムにインストールされているパッケージを次のコマンドでアップグレードさせてください。これでエラーが発生しなくなり、正常に再 生を行うよう になると思われます（2006.07.13 現在復帰済み）。

 # apt-get update
 # apt-get dist-upgrade

サウンドプレーヤーが以下へアップグレードされれば不具合は修復されます。

  • CD プレイヤー 2.14.2-*
  • Rhythmbox 楽曲プレーヤー 0.9.5-1
  • Sound Juicer CD 取り込みツール 2.14.4-2

xdm でユーザーログイン時 エディタ Gedit が起動しない（2006.04.16） (2007.02.06 既に修復済み)

ディスプレイ・マネージャに xdm を選択してユーザーログインすると、エディタ Gedit が root 権限で起動しません。（2006.04.16）
このバグは、xdm 1:1.0.5-2 では既に修復されたのでで問題ありません。

日本語キーボードの自動認識が失敗する（2006.04.16） (2007.01.26 現在修正済み)

日本語キーボードから、"」" と "]" の文字入力ができません。（2006.04.16）

[対処方法] （2006.04.22 当方確認）
日本語キーボードの自動認識によるデフォルト設定

 Generic 105-key (Intl) PC

これを正しく再設定するには、パネルメニューより　[デスクトップ] > [設定] > [キーボード] > [レイアウト] > [キーボードの形式]　で次の形式に変更します。

 Japanese 106-key

ボリューム・コントロールが起動時ミュートに固定される (2006.04.16) (2006.12.12 現在修正済み) (2007.4.25 再現性確認) (2008.02.12 現在再現性なし)

ボリューム・コントロールが、起動時ミュートに固定されます。サウンドドライバが正常に読み込まれているのなら、ボリュームのプロパ ティでミュート を解除すれば、音声が出るようになります。（2006.04.16）
起動時ミュートに固定されるのを解除するには、# alsaconf コマンドが有効であるようです。

GNOME環境のインストールが不能 (2006.05.10) (2007.01.26 現在修正済み)

いいえ。ただ、テスト版で大幅なパッケージ更新がある場合には、新規インストールや更新が一時的に不能となることがよくあります。しば らく時間を置 いてから再度実行してください。以下も参照のこと。

  • 次のコマンドは利用できません。(as of 2006.05.10) (2007.01.26 現在復旧済み) 
     # apt-get install gnome
     パッケージリストを読み込んでいます... 完了
     依存関係ツリーを作成しています... 完了
     パッケージ gnome はデータベースには存在しますが、利用できません。
     おそらく、そのパッケージが見つからないか、もう古くなっているか、
     あるいは別のソースからのみしか利用できないという状況が考えられます
     E: パッケージ gnome にはインストール候補がありません
  • 替わりに次のコマンドが利用可能です。（as of 2006.05.11） 
     # apt-get install gnome-desktop-environment
