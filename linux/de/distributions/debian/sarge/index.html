Sarge ネットワーク・インストール ('netinst') の手順について

以下の手順は、Sarge Netinst CD を使ってインストールする場合の概要(流れ)を説明したものです。

  1. sarge-i386-netinst.iso を入手します。最新版 (2006-02-04現在) は debian-31r1a-i386-netinst.iso です。iso イメージからCD-R ライティングソフトを使って CD-R を作成する方法についてはここでは省略しますので、当サイト内の関連ページをご覧ください。
  2. CD-ROM ドライブに sarge-i386-netinst CD を挿入した後、システムを再起動させます。
  3. 「Debian」というロゴとともに起動画面が表示されます。リターンキーを押してインストール作業を開始します。
    Linux kernel 2.6.8 のインストールはオプションなので、キーボードより linux26 と入力した後リターンキーを押す必要があります。
  4. 「Choose Language」の画面に切り替わったら、Japaneseｰ 日本語 を選びます。
  5. 「キーボード配置の選択」では、日本(106 キー) を選びます。そうすると、すぐにハードウェアの自動検出が開始され、インストール用コンポーネントが自動的にインストールされます。
  6. ネットワークの設定。無線LAN の場合、eth0 の無線 ESS ID および WEP キーの入力を求められます。システムがルーターと接続されているのであれば、DHCP による自動設定が実行されます。ただし、自動設定が完了できないときは、たいてい手動による再設定を実行すれば解決できます。
  7. ディスクのパーティショニングを行います。この項目では、Fedora や KNOPPIX のハードディスク・インストール時と同様、パーティションの設定を対話式に行うことになります。必要に応じて、スワップ領域とシステ ム領域を設定します。
  8. Debian ベースシステムのインストールが自動的に開始され、完了すれば次の項目へと進みます。
  9. ブートローダー GRUB をインストールするか否かを選択します。これはオプションですが、Linux を単独でインストールする場合や、Windows OS インストール済みのシステムの空き領域に Linux を追加インストールする場合は、GRUB を MBR へインストールするのがふつうです。
  10. 以上インストールが完了すれば CD-ROM はドライブから自動排出されるので、その CDメディアを取り出してからリターンキーを押して作業を続行(ここではシステムを再起動)します。
  11. Debian ベースシステムの設定を行います。
    1. 時間帯の設定 ハードウェアの時計は GMT に合わせていますか? ＜いいえ＞
    2. Asia/Tokyo の時間帯でよろしいですか? ＜はい＞
    3. アカウントのパスワード設定
    4. PPP 接続の設定 (オプション)
    5. APT の設定・ミラーサーバーの選択
    6. パッケージの選択 (tasksel による設定) GNOME/KDE によるグラフィカルなデスクトップ操作が行えるようにするには、ふつう「デスクトップ環境」を選択します。なお、始めから日 本語によるインストールを選択 している場合には、canna canna-utils language-env といった日本語入力環境に必要な最低限のパッケージも自動的にインストールされます。印刷サーバも選択しておくと良いでしょ う。
  12. インターネットに自動接続し、ミラーサイトよりパッケージのダウンロードが開始されます。デスクトップ環境が選択されている場合 には、ダウン ロードは 600MB 前後のサイズとなります。ただし、展開後には 1.7GB のディスク容量が必要となります。
  13. ダウンロードが完了したら、xserver-xfree86 の設定画面になります。この項目では、以下の設定を行います。
    1. ビデオチップの選択
    2. カーネルフレーム・バッファデバイスを使いますか？ ＜はい＞
    3. マウスの自動認識を試みますか? ＜はい＞
    4. モニタの自動認識を試みますか? ＜いいえ＞
      ※液晶ディスプレイは自動認識されにくいので、＜いいえ＞を選択しマニュアル設定することを推めます。
    5. あなたのモニタはLCDデバイスですか? LCDなら＜はい＞、CRTなら＜いいえ＞
    6. モニタの特性を選択 (Medium または Advanced)
  14. ダウンロードしたパッケージが展開され、インストールが自動的に開始されます。インストールが完了したら、次の項目が表示されま す。
  15. Exim v4 の設定 (exim4-config)
    「ローカル配信のみ； ネットワークなし」を選択します。ISP等のメールの受信、送信には通常 exim を利用しないので、この設定で問題ありません。なお、必要なければこの項目は設定しないよう選択することも可能です。
  16. 以上で Debian ベースシステムの設定は完了です。(この設定をやり直すには、 base-config を再実行します。)
  17. GNOME デスクトップマネージャが日本語で立ち上がり、ログイン画面が表示されます。先に登録した一般ユーザーでログインしてください。
  18. GNOMEデスクトップが標準で立ち上がります。この時点で、表示は全て日本語になっています。
    デスクトップの解像度が適当でない場合には、 # base-config でなくて # dpkg-reconfigure xserver-xfree86 を実行します。この設定変更を反映させる(有効とする)には、システムを再起動させる必要があります。
  19. 日本語入力に Kinput2-Canna を利用したいときは、$set-language-env を実行します。8:ja (Nihongo Japanese) を選択し、その他の質問にはリターンキーを押していけば問題なく設定が済みます。
    SCIM-UIM-Anty を利用するときは、$set-language-env を実行せず、こちらを参照のこと。
  20. 一度ログアウトし、再び同じユーザーでログインします。
  21. Shift + Space キーで日本語入力が開始できます。

    以上で Debian 3.1 (Sarge) のインストールと日本語環境の構築は完了です。

システムやパッケージの更新(アップデート)を行う（コマンドラインで操作）

デスクトップ・メニューにある「ターミナル コンソール」より、次のコマンドを使ってアップデートが行えます。


# apt-get update …システムのデータベースを最新のものに更新します
# apt-get upgrade …実際にアップデートを実行します
# apt-get dist-upgrade …全てのパッケージがアップデートの対象となります

インストール作業が完了したら、できるだけ早い機会にアップデートすることが大切です。アップデートの作業を随時行うことで、システム を最新の状態 に管理できます。

  • インタラクティブなテキストベース・ツール Aptitude も利用できます。
    Aptitude コマンドの詳細については、DebianLinux-FAQ/Aptitude 参照。

APT の sources.list 設定について

↑

3.1 Sarge のデフォルトの sources.list

ネットワークインストール直後の etc/apt/sources.list の初期設定は、次のとおりです。

#deb file:///cdrom/ sarge main

deb http://ftp.jp.debian.org/debian/ stable main
deb-src http://ftp.jp.debian.org/debian/ stable main

deb http://security.debian.org/ stable/updates main

↑

sources.list を修正して Sarge 専用で利用する（必須事項）

次のように /etc/apt/sources.list を書き換えれば、 Sarge 向けに用意されたパッケージのみがアップデートの対象となります。下記の例では、Security アップデートも対象に含まれています。いずれの場合も、stable を sarge に置き換えただけですが、この設定はユーザーが必ず行わなくてはいけません。

#deb file:///cdrom/ sarge main

deb http://ftp.jp.debian.org/debian/ sarge main contrib non-free
deb-src http://ftp.jp.debian.org/debian/ sarge main contrib non-free

deb http://security.debian.org/ sarge/updates main contrib non-free

追加したら、apt-get update と apt-get upgrade を順に実行してください。
上の例のように contrib および non-free に含まれるパッケージも扱える対象とすることが可能です。

ローカルに保存したファイルを APT-line に追加するには？

  1. Debian パッケージ用の書庫（フォルダ）を任意の場所に任意の名前で作成
    例： /tem/deb あるいは /home/hoge/deb など。
  2. Debian パッケージをそのフォルダにコピーまたは移動。
  3. /etc/apt/sources.list に次の一行を追加。
    deb file:/home/hoge/deb ./
  4. コマンドラインで、そのフォルダへ移動し dpkg-scanpackages を実行。
    $ cd deb （または $ cd /tem/deb/ などとして移動。）
    $ dpkg-scanpackages . /dev/null| gzip -9 > Packages.gz
  5. アップデートの実行。
    # apt-get update
    以上で、作成した書庫が APT-line に追加されます。

これで、今後このフォルダにあるパッケージをインストールする場合、たとえ依存関係で不足するパッケージがあっても他のリポジトリを参 照し自動的に インストールしてくれるようになります。

一時的にテキストモード (CUI) でログインする

普段グラフィカルログインで利用していて、一時的にテキストモードに切替えたいという場合は結構あり得ます。runlevel を恒久的に変更するには起動時の設定ファイルを書き換える必要があります。一時的にテキストモードを使用する際には、次の TIPS のとおり操作してください。

『ディ スプレイマネージャからコンソールに一時的に戻すことはできますか』

「xdm の場合には、Ctrl+r を押すとコンソールになります。また、xdm を含めてその他のディスプレイマネージャでは、Ctrl+Alt+F1〜F6 を押すと、それぞれの仮想コンソールを開きます。元に戻すには Ctrl+Alt+F7 を押します。」

こちらで確認したところ、Debian 3.1 (sarge) では gdm (GNOME Display Manager) の場合も、Ctrl + Alt + F1〜F6 キーでテキストモードに切り替わります。ただし、バックグランドでは既にディスプレイマネージャが起動しているようなので、startx コマンドは使えません。グラフィカルログインに切替える場合は、必ず Ctrl + Alt + F7 キーで元に戻る必要があります。

システムをシャットダウン(終了)させるには？

ユーザーはグラフィカルログイン時、直接シャットダウンさせることはできません。

Debian では Fedora や Red Hat Linux の場合と勝手が違って、グラフィカルログイン時(例えば GNOME 環境での運用時)には、ユーザーが直接シャットダウンできない仕組みを採用しています。これはセキュリティ確保のためと考えられます。

具体的には、一度ログオフし、ディスプレイマネージャー (xdm: X Display Manager) から今度は root ログインした後で「シャットダウン」の項目が選択できるようになります。
ただし、ディスプレイマネージャーに gdm (GNOME Display Mamnager) を使用した場合には、ログイン画面で root 権限のシャットダウンが可能となるので、この方が実際便利でしょう。「GNOME デスクトップマネージャ」のメニューから アクション ＞ 停止 の項目を選択してシャットダウンさせます。

(2005.06.06 現在、Sarge は GNOME 2.8 に置き換えられていますが、デフォルト画面の基本的な配置は変わりません。)

↑

フロッピーや USB メモリ、CDのデータを読み込む方法は？

GNOME 2.8 の採用によって、従来の多くの Linux ディストリビューションで一般的であった「ドライブのマウント」という作業を特に意識する必要はありません。デスクトップに用意されてい る「コンピュー タ」アイコンをマウスでダブルクリックして開くことができます。

特に、USB メモリと CD-ROM の場合、デフォルトは自動的にファイルを開きます。
外部メディアを取り出すときは、Windows の場合と同様に「メディアの停止」、つまりアンマウントいう作業が必要です。Device is Busy と表示されアンマウント不能な時は、lazy オプション（-l）で対処可能です。

 # umount /media/usbdisk -l
  • [注意] ただし Linux kernel 2.4 をインストールした時には「ボリューム・マネージメントツール」(自動マウント機能) が使えません。Linux kernel 2.6 をインストールした時のみ有効です。

ファイルの操作を「シングルクリック」で済ませたい

Windows 標準では、アイコンの「ダブルクリック」でファイルが開きます。一方、Linux の場合、GNOME 環境ではダブルクリック、KDE 環境ではシングルクリックが一般的な初期設定のようです。

GNOME 環境でシングルクリックへと設定変更するには、デスクトップの「ここからスタート」または「ホームのアイコン」 ＞ デスクトップの設定 ＞ ファイル管理 ＞ 「動作」タブ ＞ 「シングル・クリックでアイテムを起動する」 にチェックを入れてください。

クイック起動アイコンの大きさを変更する

タスクバー上でマウスの右ボタンをクリックし、プロパティから サイズの項目で設定を変更します。初期設定では、サイズが24ピクセルになっています。この数字を36くらいにすると Red Hat 9、Fedora Core Linux で採用されている GNOME とほぼ同じ見栄えになるでしょう。

画面最上部のタスクバー (Top Panel) を消して画面を広く使う

まず、画面最下部にあるパネル (Bottom Pnel) の上でマウスを右クリックして、「このパネルを削除」を選択／実行します。次に、画面最上部にあるパネル (Top Panel) 上でマウスの右ボタンをクリックし、向きを「上」→「下」へ変更します。

これでクイック起動一覧が画面最下部にすっきりと並びますが、このままではデフォルトにあった2つのボタンが消えてしまうので、以下の 手順で元通り 復元します。

  • [ウインドウの一覧]

    パネル(タスクバー) を右クリック ＞ パネルへ追加 ＞ ユーティリティ ＞ ウインドウの一覧 をクリックすれば、追加できます。
  • [ワークスペース切り替え器]

    パネル(タスクバー) を右クリック ＞ パネルへ追加 ＞ ユーティリティ ＞ ワークスペース切り替え器 をクリックすれば、追加できます。

Firefox 1.5.x バージョンを安定版（Sarge）で使うには？

Debian 公式の安定版向けパッケージには、Firefox 1.5 は含まれていません（2006.04.05 現在）。不安定版や実験版から借用するのは、関連パッケージの変更を伴う心配から推奨できません。
そこで、Firefox 開発元の公式サイトより Linux用パッケージを入手し次の手順で起動させます。

 $ tar -xzvf firefox-1.5.0.1.tar.gz
 $ ./firefox/firefox

なお、Mozilla-firefox 1.0.4 が既にインストールされている環境では、これを削除してから Firefox 1.5 のインストールを開始してください（mozilla-firefox-locale-ja がインストール済みなら、これも自動的に削除されます）。

 # apt-get remove mozilla-firefox

firefox-1.5.0.1.tar.gz を firefox 以下のディレクトリ展開するだけの作業なので、firefox-1.5.0.1 を削除したいときはこのディレクトリごと「ごみ箱」へ移動させます。

  • 展開先がユーザーディレクトリ直下の場合、起動コマンドは ./firefox/firefox となります。firefox で起動させるにはシンボリック・リンクを張る必要があります。
  • デスクトップのランチャーに登録すれば、素早く起動できます。アイコンは、展開先ディレクトリに用意されています。
  • firefox-1.5.0.1.tar.gz を # alien firefox-1.5.0.1.tar.gz で Deb 形式に変換しても、通常の Dpkg -i hoge コマンドでインストールすることは無理です。
