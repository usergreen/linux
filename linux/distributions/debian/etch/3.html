日本語入力環境の設定 (uim-Anthy) -2008.01.02 更新

uim-Anthy は標準でインストールされます。したがって、何も追加設定を行う必要はありません。ただし、最小構成でインストールした場合は、 uimu、uim- anthy などを任意に追加インストールします。

  • 初期設定では [全角/半角] キーで日本語入力に切替えます。

日本語入力環境の設定 (SCIM-Anthy) -2008.01.02 更新

Etch では、日本語入力環境の設定が Sarge の場合より簡単になっています。
$ set-language-env コマンドを使用したり、‾/.xsessionの設定ファイルを作成したりする作業が不要となりました。これは、im-switch が新規導入されたためです。

次のコマンドを実行してください（パッケージの依存関係変更により、実行コマンドが簡略化されました）。

 # apt-get install scim-anthy kasumi

im-switch の紹介と概要

「im-switchは、FedoraCore?で 使われている入力メソッドフレームワークを部分的にDebianに移植したものです。

Debianでは多様なバックエンド、入力メソッド(XIMやIMmoduleなど)がありますが、その代償として「デフォルト」がな く、各ユー ザーが手動で環境変数や‾/.xsessionを編集しなければなりませんでした。

im-switchのフレームワークに則った入力メソッドパッケージはim-switch管理下のalternativeの1つとな り、これによっ て優先度の高い入力メソッドパッケージが自動的に「デフォルト」の入力メソッドになります(im-switchで変更可能)。また、デ フォルトとは別の入 力メソッドとは違うものを使いたいユーザーも、im-switchコマンドを使って変更できます。」

  • https://members.fsij.org/trac/codefest2005/wiki/kmuto:im-switch
  • [debian-devel:16196] proposal: im-switch framework
  • http://packages.debian.org/unstable/x11/im-switch
    im-switch provides the framework to switch default of input method on X Window System. Input method is usually needed by Japanese, Chinese and Korean to input their native language characters.
    By installing this package, input method, and conversion engine, you can use input method (XIM and IMmodule) soon without any more configuration. You can also choose your favorite input method by 'im-switch' command.

language-env による設定

Debian Etch でも、Debian Sarge 以前の場合同様、 set-language-env コマンドを使った日本語設定が可能です。kinput2-canna で日本語入力設定したいとき利用できます。

 $ set-language-env

ただし一般的には、im-switch が非常に上手く出来ているので、im-switch で設定することを推奨します。

使い方: set-language-env [オプション]

-l 言語 : 言語を指定する (なければメニューから選ぶ)

-h : このヘルプメッセージ

-v : 実行時の詳しい情報を表示

-s : サポートされている言語の一覧を表示して終了する

-r : 全ての設定を消去する

-N : 別の set-language-env を fork しない (内部で使用する)

-c : 言語ごとの文字 (漢字など) を使わない (内部で使用する)

-C : 言語ごとの文字 (漢字など) を使う (内部で使用する)

-E : /etc/skel ディレクトリを設定する (ルートユーザーのみ)

[Enter] キーを押してください。終了します。
