2006-04-26 bug report

xkb-data をインストールすると、X でユーザーログインが不能となる可能性があります。
xkb-data は複雑な依存関係のため、一度インストールすると単独では削除ができません。
# apt-get remove xkb-data
パッケージリストを読み込んでいます... 完了
依存関係ツリーを作成しています... 完了
以下のパッケージは「削除」されます:

  gnome gnome-applets gnome-control-center gnome-core
  gnome-desktop-environment gnome-office gnome-panel gnome-session
  gnome-terminal libxklavier10 nautilus xkb-data

アップグレード: 0 個、新規インストール: 0 個、削除: 12 個、保留: 0 個。
0B のアーカイブを取得する必要があります。
展開後に 12.4MB のディスク容量が解放されます。
続行しますか [Y/n]? n
中断しました。

したがって、問題発生時は次のコマンドで修復を行なってみるようお薦めします。

# dpkg -r --force-depends xkb-data
dpkg: xkb-data: 依存関係に問題があります。しかし要求に従い削除しています:

 libxklavier10 は以下に依存 (depends) します: xkb-data (>= 0.8) | xkb-data-legacy ...しかし:
  パッケージ xkb-data を提供していますが、削除しようとしています。
  パッケージ xkb-data-legacy はインストールされていません。

(データベースを読み込んでいます ... 現在 55900 個のファイルとディレクトリがイン ストールされています。)
xkb-data を削除しています ...

当方では、以上でこの不具合を修復し、ほぼ直前の状態に復帰させました（以前の状態へ完全に修復はできていません）。
