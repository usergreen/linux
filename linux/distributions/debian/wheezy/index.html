Debian Wheezy をインストールするには?

　Debian インストーラ 7.0 リリース候補 (Release Candidate) 3 がリリースされたので、これを使ってください。http://www.debian.org/devel/debian-installer /News /2013/20130502
　Wheezy RC1（リリース候補第1版）のインストーラを使うことが推奨されています。
　次の公式ページを見てください：http://www.debian.org/devel/debian-installer/News/2013/20130217
　なお、このリリースでは既知の問題が確認されているので、詳細や既知の問題の完全な一覧については正誤表を 見ること が必要です。
　インストール時には、http://wiki.debian.org/DebianInstaller/Today というページも参照する必要があります。

ネットワーク・インストールするには？ （2014年4月12日更新）

　Wheezy には、フリーでないファームウェアは含まれなくなったので、容易にネットワーク・インストールをするためには、コツが要ります。フリーでない ファームウェ アが入ったインストール用イメージも配布されているので、これを利用すると良いでしょう。
　配布先：http://cdimage.debian.org/cdimage/unofficial/non-free/cd-including-firmware/7.4.0/i386/iso-cd/
　配布先：http://cdimage.debian.org/cdimage/unofficial/non-free/cd-including-firmware/wheezy_di_beta4/i386/iso-cd/

Wheezy に対応するファームウェア全般の情報が知りたい。

　次の Wikiページを参考してください：http://wiki.debian.org/Firmware

インターネットに接続出来ないのです が、どうし たらインストールできますか？

　CD あるいは DVD 用の iso イメージからインストール用のディスクを作成し、インストール完了後、ネットワーク接続の設定を行うことでこの問題を回避することが可能です。

Wheezy の sources.list は？

/etc/apt/sources.list という設定ファイルには、ミラーサイトを選び以下のように記述を行います。

deb http://ftp.jp.debian.org/debian/ wheezy main contrib non-free
deb http://security.debian.org/ wheezy/updates main contrib non-free

Wheezy の標準（デフォルト）のデスクトップ環境は何ですか？

Debian インストーラーによる標準的なインストールでは、GNOME 3 が自動的に選択されます。GNOME バージョン 3 に特有な FAQ や Tips は、GNOME 3 のページをご覧ください。
もちろん、KDE や Xfce などを選択し利用することも可能です。

Iceape Composer が見つかりません。

１つ前の安定版では、Composer (Mozilla Composer) が提供されていましたが、Wheezy の公式版ではリリースされませんでした。しかしながら、Mozilla プロジェクトの本家からは、現在 Linux 用の日本語版の配布が行われるようになったので、これを利用することが可能です。

2014年4月時点、Composer は Seamonkey Composer という名称に変わりましたが、実際の中身は従来の Composer そのものを継承しています。

NHK など一部のサイトで動画再生や音声が出ないのですが？

再生に必要となる Flash Player や動画関連のライブラリーが最新版に対応していないことが主な原因と考えられます。数年前は問題なく再生されたのに、ある日以降から急に利用できなくなっ た場合は、恐らくこれが原因でしょう。「メディアエラー メディアの読み込みができません」と表示されることもあります。インターネット上で提供される NHK 関連のラジオニュースやネットラジオ、聞き逃し番組で発生することが報告されています。

ディストリビューションがサポートする最新版へアップデートするか、もしくは最新版を新規インストールするのが最も確実な解決法です。状況が許す のであれば、i386 をインストールしている場合は AMD64(IA64) へ移行すべきです。サイト運営者側が Linux の IA32(i386) アーキテクチャーに対応していなかったり、古いバージョンの Flash Player をサポートしないことで制約を受けている可能性があります。ユーザーが各種の設定ファイルを変更して対処しようとしても、簡単には解決出来ないようです。

なお、Flash Player が動作するシステム要件として、Firefox あるいは Google Chrome の最新版が必要とされています。（出典：http://www.adobe.com/products/flashplayer/tech- specs.html ）
