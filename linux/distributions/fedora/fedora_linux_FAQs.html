4.APT for RPM (APT-RPM)導入の仕方について

１．APTのRPMパッケージの入手

　Fedora Projectのウェブサイト[[FedoraHOWTO:http://www.fedora.us/wiki/FedoraHOWTO]]からファイル名「apt-0.5.15cnc1-0.fdr.3.1.i386.rpm」（Fedora Core 1の場合）を任意の場所にダウンロードします。（ここでは、ルート・ディレクトリーにダウンロードしたものとして、以下説明します。）ダウンロードサイズは、789KBです。

２．APTのRPMパッケージのインストール

　まず、コンソールからrootでログインします。既にユーザーログインしている場合には、 su（スーパーユーザー） でルートログインするためにパスワードを入力します。ダウンロードしたファイルは、RPMでパッケージされていますので、インストールはコンソール画面から次のように入力します。入力が面倒な方は、次のコマンドを予めコピーしておき、それをコンソール画面に貼り付けたら楽でしょうし、入力間違いも防げます。

rpm -ivh apt-0.5.15cnc1-0.fdr.3.1.i386.rpm

　入力したらリターンキーを押します。自動的にインストールが開始されると次の例のように表示され、数秒の内に完了します。

[user@localhost xxxx]$ su

Password: xxxxxxxx

[root@localhost xxxx]# rpm -ivh apt-0.5.15cnc1-0.fdr.3.1.i386.rpm

警告: apt-0.5.15cnc1-0.fdr.3.1.i386.rpm: V3 DSA signature: NOKEY, key ID 8df56d0 5

Preparing... ########################################### [100%]

1:apt ########################################### [100%]

３．APTリストの更新

　APTのインストールが無事に済めば、次に行う作業は最新のリストにアップデートする作業です。コンソール画面から、次のコマンドを入力します。

apt-get update

入力したらリターンキーを押します。回線の込み具合によっては少し時間がかかります。

[root@localhost xxxx]# apt-get update

You don't seem to have one or more of the needed GPG keys in your RPM database.

Importing them now...

Get:1 http://download.fedora.us fedora/1/i386/os pkglist [1445kB]

Get:2 http://macromedia.mplug.org fedora/1/macromedia pkglist [907B]

(---途中経過省略---)

Reading Package Lists... Done

Building Dependency Tree... Done

４．アップグレードの実行

　インターネットの配布先から、最新のRPMパッケージをダウンロードし、インストールするためには、引き続いて次のように入力します。

apt-get upgrade

　入力したらリターンキーを押します。公開されたばかりで、更新すべきパッケージがあまりないので、作業はすぐに完了します。

[root@localhost xxxx]# apt-get upgrade

Reading Package Lists... Done

Building Dependency Tree... Done

The following packages will be upgraded

perl-DateManip

1 upgraded, 0 newly installed, 0 removed and 0 not upgraded.

Need to get 157kB of archives.

After unpacking 82.8kB of additional disk space will be used.

Do you want to continue? [Y/n] y

　途中、上の例のように作業を続けるか取消すかを尋ねてくるので、ここでは「y」を入力しリターンキーを押します。すると作業が再開され、次のような画面が表示されます。経過表示が１００％になれば、作業終了です。

Get:1 http://download.fedora.us fedora/1/i386/stable perl-DateManip 0:5.42-0.fdr .2.a.1 [157kB]

Fetched 157kB in 2s (59.6kB/s)

Checking GPG signatures...

Executing RPM (-Uvh)...

Preparing... ########################################### [100%]

1:perl-DateManip ########################################### [100%]



５．APTコマンドについて

　以上の一連の手順で、パッケージ管理はすでにAPTで行えるようになりましたので、今後はAPTリストの更新とアップグレードの実行を随時繰り返すだけです。APTで使用する代表的なコマンドを参考までに載せておきます。基本となるのは次のコマンドです。

#apt-get update　・・・・・・・・・・・・最新のリストに更新する

#apt-get upgrade　・・・・・・・・・・・・更新されたリストに従って実行をする

#apt-get dist-upgrade　・・・・・・・・・OS（カーネル）自体の更新を実行する

#apt-get install 〔パッケージ名〕・・・・個別に指定してインストールする

#apt-get remove 〔パッケージ名〕・・・・・個別に指定して削除（アンインストール）する

#apt-get clean　・・・・・・・・・・・・・一時保存されているキャッシュファイルを削除する



5.グラフィカル管理ツール：Synaptic の導入について

　Synaptic と呼ばれる GUI フロントエンドを使って APT の操作ができます。詳しくは、下の項目を参照してください。



6.Synaptic とは何ですか

APT の[[グラフィカルな管理ツール:http://www.h7.dion.ne.jp/~greens/Fedora_apt-for-rpm/Screenshot-Synaptic.jpg です。Fedora Core 1に対しても提供が開始されました。

Synaptic を導入するには、上記のサイトに挙がっている synaptic-0.45-1.1.i386.rpm をダウンロードし、インストールします。もしくは、コンソールから次のコマンドを入力します。

apt-get install synaptic

インストール後、Synaptic を起動させるには、次の２つの方法があります。

メインメニュー（赤帽アイコン）＞ システム設定 ＞他のシステム設定 ＞Synaptic パッケージマネージャ　とマウスで選択し、クリックします。

コンソールから次のとおり入力し、実行します。

Synaptic



7.アップデートはどうやったら良いのですか？

新しいパッケージの追加やセキュリティ、バグの更新に関しては、次のサイトからダウウンロードできます。

http://fedora.redhat.com/download/updates.html

上に述べたAPTの利用のほか、従来のRed Hat Update Agent (up2date)も利用できます。



8.バグ・セキュリティに関する重要な情報はどこで手に入りますか

Fedora の fedora-announce-list というメールリストに加入すると便利です。その都度、無料でメールによる配信を受けることができます。

過去の情報は、こちらのアーカイブ http://www.redhat.com/archives/fedora-announce-list/index.html に保管されていています。メールリスト(Mailing Lists)の紹介（英文）はこちら:http://fedora.redhat.com/participate/communicate/ にあります。

fedora-announce-list を含め、次にあげるメールリストが利用できます。これを購読するためには、件名に subscribe と書いて <希望するリスト名>-request@redhat.com 宛てにメールで申込んでください。



-fedora-announce-list - Announcements of Fedora Core changes and events

-fedora-list - For users of Fedora Core releases

-fedora-test-list - For testers of Fedora Core test releases

-fedora-devel-list - For developers, developers, developers

-fedora-docs-list - For participants of the docs project

-fedora-desktop-list - For discussions about desktop issues such as user interfaces, artwork, and usability

-fedora-config-list - For discussions about the development of configuration tools

-fedora-de-list - For discussions about Fedora in the German language

13.かな漢字変換がおかしい。辞書が壊れているのですか

原因がはっきりとしないようですが、初期設定の状態では、漢字変換の精度に欠ける場合があります。対処方法として次の方法が考えられます。

◯ デスクトップあるユーザーフォルダのアイコンをダブルクリックし、メニュー　＞編集　＞設定　の項目から、デフォルト表示の「隠しフォルダを表示する」にチェックを入れたら.cannaのファイルが表示されるので、ファイル名を変更し、保存します。設定の変更を反映するためには、ログインし直します。

◯ /.canna の "fuzokugo" のコメントを外します。

◯ Cannaを別のものに置き換えます。フリーで利用できる優秀なかな漢字変換ソフトがあります。
