fedora.us とは何ですか？ 

Fedora.us とは、レッドハット・リナックスや Fedora Core 向けに高品質な追加パッケージを開発する、コミュニティー主体の信頼すべきプロジェクトです。このコミュニティのボランティアが目指す パッケージの開発 は、デビアンのコミュニティが目標としている理想と大変似たところがあります。

↑

Fedora.us と fedora.redhat.com の Fedora Project との違いは何ですか。 

fedora.redhat.com で取り組んでいる Fedora Project は、レッドハット社が中心となって行っているコミュニティーを主体としたディストリビューション開発のためのプロジェクトです。従来レッ ドハットリナック スと呼ばれていたものは、現在フェソーラプロジェクトと呼ばれることになりました。このプロジェクトでは、長期的にみれば、デビアンで行 われているような 手法で、ボランティアの開発者がリナックス用のディストリビューションの開発を目指しますが、開発を迅速に進めるようにします。2003 年9月、 fedora.us に関わっているメンバーは、レッドハットリナックスと併合して、Fedora Project という新たな組織を設立することに合意しました。こうして、技術者やコミュニティの熱心な協力者が一体となって、ディストリビューション 全般に関する品質 の向上をめざします。

現在のところ、2004年4月に公開予定の Fedora Core 2 の正式なリリースを控えて、両者のプロジェクト併合が急速に進展しています。プロジェクトが完全に併合された時点で、fedora.us が提供するパッケージは、一体化されたフェドーラプロジェクトの公式な一部の、Fedora Extra として管理されることになっています。

また同時に、fedora.us は、独立した一プロジェクトとして存続することとされています。当プロジェクトが採用している品質検査の手法は、最も高品質なパッケージ を提供することに 努めるためのものです。Fedora.us のリポジトリーは、今後ともレッドハットリナックス8.0 や 9、あるいは Fedora Core 1 用の追加パッケージや、アップデートのためのパッケージを提供することを目的としています。一方、Fedora Legacy という名称のプロジェクトでは、レッドハット社がもはやサポートしない旧バージョンに対して、セキュリティアップデートを提供します。

どうしてfedora.us を使うのですか。 

現行のレッドハット・リナックスは、極めて容易にインストール出来ますが、ソフトウェアを適切に追加するには、時間がかかるだけでな く、時には困難 な作業です。慣れた人ならやり方を心得ているでしょうが、それでもなお、ソフトウェアの手作業によるビルドやインストールは、時間のかか ることです。新規 ユーザーにとっては、パッケージが入手不可能な時や、現行のパッケージに互換性が無い場合、大変苦労することになります。フェドーラで は、ソフトウェアの 追加がとてもやさしいので、時間の節約になります。たいていの場合は、良さそうだと思ったパッケージをフェドーラのカタログから選ぶだけ で済みます。時に は経験者の助けが必要となるかもしれません。このような場合に、何時間もかけてタールボールをビルドし、インストールするよりも、フェ ドーラのサイトから インストールすれば、解決できます。その結果、経験のあるユーザーに助けを求めることになりかねませんから、結局、全員が貴重な時間を割 くことになりま す。

さらに、fedora.us は、Fedora Extra を扱う信頼に足る権威ある開発プロジェクトです。 Fedora.us は、レッドハットリナックスや フェドーラコア向けの最も高品質な追加パッケージを提供するために存在し、今後もそのために活動します。

フレッシュRPMなどその他のリポジトリーとフェドーラとはどこが違うのですか？ 

フレッシュRPMなど、他のリポジトリーは、全て一個人が作成したものです。フェドーラがより高品質で数多くのパッケージを管理できる のは、共通の 目的を持って開発するたくさんの研究者を抱えているからです。お互いが協力し合ってリポジトリー用のパッケージ開発を進めることで、より 一層高品質なもの を実現でき、パッケージ同士が「クラッシュ」してしまう危険性を少なくすることにつながると考えています。Fedora.us のリポジトリーとクラッシュする恐れがなくて、なおかつ、私達のプロジェクトと協調して高品質なパッケージを提供しているサイトが存在し ます。その一例と して、rpm.livna.org や jpackage.org があります。

フェドーラ プロジェクトは、誰がいつ始めたのですか？ 

先の「フェドーラ　リナックス　プロジェクト」は、2002年12月、ハワイ大学コンピューターサイエンス学部に在籍する学生 Warren Togami 氏によって創設され、レッドハットリナックス用のプラットフォームに適したサードパーティ製のソフトウェア開発を目的とする、国際的なボ ランティア組織で す。

統合される新組織においては、開発過程によりいっそう大きなコミュニティーが参画することで、一般の多くの人々に利益をもたらすことに なるでしょ う。
