Red Hat Linux について

レッドハット・リナックスは、以前 red hat 社からリリースされたもので、現在その開発は終了しています。ただしその実質的な開発は、 red hat 社と Fedora Linux Prfoject が併合して新たに設立された Fedora Project へと引き継がれています。詳しくは、Fedora Linux のページを見てください。

たとえば、Red Hat Linux の最終リリースにあたる Red Hat Linux 9 については、red hat 社の公式なサポート(アップデート・リリースの提供など)が、2004年4月30日限りで終了してしまいました。しかしながら、コミュニティー主体で取り 組まれている Fedora Extras Project (Fedora Linux Prfoject から改称) や Fedora Legacy Project からアップデート・リリースの提供を受けることが可能です。

Red Hat Linux 9 で APT を利用するには？

Using Fedora Legacy's apt for Red Hat Linux 9

Red Hat Linux 9 でレガシー・サポートを利用するための設定方法

apt is an automated package management program developed for Debian Linux which may be used to install, remove and update packages on an RPM based system. It will help you to keep your system up to date easily.

APT というのは、もともと Debian Linux 用に開発された、パッケージ管理の自動化プログラムのことで、RPM のもとでシステム管理されているパッケージのインストールや削除、アップデートのために利用されます。APT を利用すれば、お使いのシステムの更新が簡単に出来ます。

NOTE: The Fedora Legacy apt program for Red Hat Linux 9 has not yet been completed. The instructions below cover installing the fedora.us version of apt. When the Fedora Legacy version is released, these instructions will be updated to reflect that fact.

注意）Fedora Legacy が提供予定の Red Hat Linux 9 用 APT は、現在のところ未完成です。以下の作業手順は、fedora.us 版の APT を利用する方法を述べています。Fedora Legacy 版がリリースされ次第、それに基づいた手順に更新されます。

Step 1: Preliminaries

ステップ1.導入前の準備

Linux prevents ordinary users from installing, removing, or modifying system software, libraries, and important configuration information. So you must have root access to proceed. You may either login as the root user, or use the su (or sudo) commands to become the root user on the machine.

Linux では、一般ユーザーがシステムのプログラムや設定情報などをインストールしたり、削除すること、あるいは改変することを許可しません。そのため、ルート権 限で作業する必要があります。ルート・ユーザーでログインするか、su（あるいは sudo）コマンドを用いてルート権限を取得するか、いづれかの方法を取ってください。

Note: Be careful when running as root! Be sure to logout of the root account as soon as you are done. Running as root is dangerous, and should only be used when needed. Typos or mistakes can destroy your system or your data, so it is important that you be careful when running as root.

注意）管理者(root)権限で運用する際の注意点。作業が終了したら、すぐにログアウトしてください。root で運用することは危険を伴うので、必要な場合以外は利用すべきではありません。操作を誤るとシステムやデータの破壊につながるので、その運用には充分気を 付けてるようにしてください。

When you are running as root, your prompt will be changed to the # character. In the command examples below, we include this prompt, however you should not type the # character when entering a command!

root で運用している際には、入力画面で # という記号が表示されます。下記の説明でこの表示をしているからといって、コマンドの入力時「#」という記号を入力してはいけません。

Step 2: Install apt

ステップ2：　APTのインストール

To install apt, use the following command as the root user on your machine:

APT をインストールするには、使っているパソコンでルート・ユーザーとなり、次のコマンドを実行します。

 # rpm -ivh http://download.fedora.us/fedora/redhat/9/i386/RPMS.stable/
 apt-0.5.5cnc6-0.fdr.8.rh90.i386.rpm

Step 2.1: Configure apt

ステップ2-1：APTの設定

Next you should configure apt to use the Fedora Legacy repositories from the official download.fedoralegacy.org site. Edit the file /etc/apt/sources.list and insert the following line at the start of the file:

次の作業は、Fedora Legacy の公式なダウンロード・サイトのリポジトリーが利用できるように、APT を設定することです。 /etc/apt/sources.list　というファイルをエディターで編集して、次の記述をファイルの冒頭に挿入してください。

 rpm http://download.fedoralegacy.org/apt redhat/9/i386 os updates legacy-utils

You will probably want to remove or comment out any other lines in /etc/apt/sources.list which start with rpm or rpm-src as these would cause apt to use non-FedoraLegacy? packages.

/etc/apt/sources.list ファイルに存在する記述の中で、 rpm or rpm-src で始まる他の部分は、削除するかコメントする必要があります。なぜなら、そうした記述は、FedoraLegacy?　 以外のパッケージを参照することになるからです。

Step 2.2: Import the Fedora Legacy Key

ステップ2-2：Fedora Legacy キーのインポート

You should import the Fedora Legacy GPG key by executing the following command:
Fedora Legacy で使われる GPG キーのインポートは、次のコマンドを使って行います。

 rpm --import http://www.fedoralegacy.org/FEDORA-LEGACY-GPG-KEY

Step 3: Update your system　ステップ3： システムのアップデート実行

Once you have installed the apt package, you should run the following commands as the root user on your system to update your system.

APT のインストールが完了したら、システムを更新(アップデート)するためには、ルートユーザーになって次のコマンドを実行します。

Step 3.1: Resynchronize the package index files from their sources

ステップ3-1:パッケージのインデックス・ファイルとソースを同期させるために、次のコマンドを実行します。

 # apt-get update

Step 3.2: Have apt update your packages

ステップ3-2: パッケージのアップデート

 # apt-get upgrade

Step 3.3: Optionally have apt clean up downloaded files

ステップ3-3: ローカルにダウンロードされたファイルの削除(オプション)

 # apt-get clean

Step 4: Subscribe to fedora-legacy-announce

ステップ4：fedora-legacy-announce メーリングリストの購入

You may subscribe to the fedora-legacy-announce mailing list to be informed by e-mail when new updates become available. This step is optional, but highly recommended.

アップデートがリリースされる度に fedora-legacy-announce というメーリングリストから電子メールによる通知を受け取ることが可能です。この選択はオプションですが、ぜひ利用することをお推めします。

Step 5: Please help us with our service!

The Fedora Legacy project is always in the need of helping hands. Please check the Participate section of our website to see what you can do to help us. As we're a community project, our success will heavily depend on helping hands - possibly you!

If you find a problem with an update published by The Fedora Legacy Project, or in The Fedora Legacy Project documentation, please let us know!
Step 6: Optionally learn additional features of apt

Below is a summary of some of the more advanced features of apt for those who wish to know more. You do not need to know these commands to keep your system updated; they are simply provided for those who want to learn more about using apt to its fullest.

  • apt-get update
    Synchronize your local apt configuration and database with the current information available on download.fedoralegacy.org and/or the mirror(s) you selected. Always execute this before starting any other work with apt to make sure your apt database is current.
  • apt-get upgrade

    Upgrade all currently installed RPMs to the latest version available of the same package, if possible. This will not remove any already installed packages or add any addition (not already installed) packages, which may mean that it fails to upgrade packages if their dependencies have changed.

    Unlike the dist-upgrade option below, this will not install new packages which obsolete packages installed on your machine or install new dependency packages.
  • apt-get dist-upgrade

    Have apt try to upgrade all packages on your system, including replacing obsolete packages with their replacement packages, and installing any needed dependencies for packages added, upgraded, or replaced.
  • apt-get install <packagename>

    Install the package named packagename, including all unmet dependencies. You will be prompted to confirm the installtion.
  • apt-get remove <packagename>

    Remove (uninstall) the package named packagename from your system.
  • apt-cache search <word>

    Search all entries in the locally cached apt repositories for a package whose name or contents match <word>.
  • apt-cache show <packagename>

    Show basic information about the package named packagename.
  • apt-get clean

    Have apt clean up downloaded RPM files to free up disk space.

※原文へのリンク　http://www.fedoralegacy.org/docs/apt-rh9.php

メーラー Sylpheed をインストールする

APT コマンドを使ってインストールします。

 # apt-get install sylpheed
 Reading Package Lists... Done
 Building Dependency Tree... Done
 The following NEW packages will be installed:
   sylpheed
 0 packages upgraded, 1 newly installed, 0 removed and 0 not upgraded.
 Inst sylpheed (0.8.9-3 Fedora Linux:9/stable)
 Conf sylpheed (0.8.9-3 Fedora Linux:9/stable)
  • ワーク ステーション タイプでインストールを行なった場合は、Sylpheed 公式サイトから ソースRPMとして提供されている sylpheed-0.9.99-1.src.rpm を利用することも可能です。コンソール(ターミナル)より次のコマンドを実行します。 
     # rpmbuild --rebuild sylpheed-0.9.99-1.src.rpm
     .....................(途中経過省略)
     Processing files: sylpheed-debuginfo-0.9.99-1
     書き込み中: /usr/src/redhat/RPMS/i386/sylpheed-0.9.99-1.i386.rpm
     書き込み中: /usr/src/redhat/RPMS/i386/sylpheed-debuginfo-0.9.99-1.i386.rpm
    sylpheed のRPMパッケージは /usr/src/redhat/RPMS/i386/ ディレクトリに作成されたので、今度はこのディレクトリに移動し、通常のインストールのコマンドを実行します。 
     # cd /usr/src/redhat/RPMS/i386/
     # rpm -ivh sylpheed-0.9.99-1.i386.rpm

ブート時のランレベルの変更方法について

ブート時にデフォルトのランレベルを変更する手順です。
http://www.jp.redhat.com/manual/Doc9/rhl-rg-ja-9/s1-grub-runlevels.html

一時的にランレベルを変更したい場合は、次のようにします。

  1. ブート画面でキーボードから a を入力します
  2. kernel オプションの画面に切り替わります
  3. 1文字分のスペース ＋ 希望のrunlevelの数字(1-6)を入力しリターンキーを押します。 
     grub append> ro root=LABEL=/ 3

RHL 7.2、RHL 8.0 のサポート停止について

2004.09.25 現在、RHL 7.2 と RHL 8.0 のサポートが停止されています。これが一時的な措置か、恒久的な措置であるのかは不明です。次のとおり告知があるので注意が必要です。

Please note that support for RHL 7.2 and RHL 8.0 has been suspended, and that new updates are no longer being generated for these versions! (http://www.fedoralegacy.org/updates/)

以前のリリース版から、より新しいリリース版へバージョンアップできますか。

パッケージ管理ツールの APT や YUM を利用することでバージョンアップができます。この話題については、fedora-legacy-list というメーリングリストに詳細があります。あるバージョンによっては問題が発生することもあるようなので、注意が必要です。

Red Hat Linux から Fedora Core の各リリース版へのバージョンアップを行うことも可能です。Fedora Core インストールCDには、クリーン・インストール以外に「アップデート・インストール」の項目が選択できるようになっています。APT や YUM 経由でのバージョンアップを利用することもできます。
