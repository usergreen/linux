国内高速ミラーサーバーを利用するには？ (YUM編)

  • SL 3.0.5 の場合 (利用しないサーバをコメント(#)しておき、国内ミラーを追加します。) 
     [sl304server]
     name=SL 305 main
     #baseurl=ftp://ftp.scientificlinux.org/linux/scientific/305/$basearch/SL/RPMS/
     baseurl=ftp://ftp.riken.jp/Linux/scientific/305/$basearch/SL/RPMS/
     
     [sl304errata]
     name=SL 305 errata
     #baseurl=ftp://ftp.scientificlinux.org/linux/scientific/305/$basearch/errata/SL/RPMS/
     baseurl=ftp://ftp.riken.jp/Linux/scientific/305/$basearch/errata/SL/RPMS/
  • SL 3.0.4 の場合 (利用しないサーバをコメント(#)しておき、国内ミラーを追加します。) 
     [sl304server]
     name=SL 304 main
     #baseurl=ftp://ftp.scientificlinux.org/linux/scientific/304/$basearch/SL/RPMS/
     baseurl=ftp://ftp.riken.jp/Linux/scientific/304/$basearch/SL/RPMS/
     
     [sl304errata]
     name=SL 304 errata
     #baseurl=ftp://ftp.scientificlinux.org/linux/scientific/304/$basearch/errata/SL/RPMS/
     baseurl=ftp://ftp.riken.jp/Linux/scientific/304/$basearch/errata/SL/RPMS/
  • SL 3.0.3 の場合 (利用しないサーバをコメント(#)しておき、国内ミラーを追加します。) 
     [sl303server]
     name=SL 303 main
     #baseurl=ftp://ftp.scientificlinux.org/linux/scientific/303/$basearch/SL/RPMS/
     baseurl=ftp://ftp.riken.jp/Linux/scientific/303/$basearch/SL/RPMS/
     
     [sl303errata]
     name=SL 303 errata
     #baseurl=ftp://ftp.scientificlinux.org/linux/scientific/303/$basearch/errata/SL/RPMS/
     baseurl=ftp://ftp.riken.jp/Linux/scientific/303/$basearch/errata/SL/RPMS/

設定を変更し上書き保存したら、コンソールから次のコマンドを実行します。

 # yum list update (yum check-update コマンドでも同じ)

ヘッダ情報がダウンロードされるので、しばらく時間がかかります。そして、

 # yum update

これで全てのパッケージが更新され、システムは最新の状態となります。

国内高速ミラーサーバーを利用するには？ (APT編)

  • SL 3.0.4 の場合 
     #apt-rpm pointing to SL
     #rpm ftp://ftp.scientificlinux.org//linux/scientific/ 304/i386/apt-rpm os updates contrib
     #rpm ftp://ftp.scientificlinux.org//linux/scientific/ 304/i386/apt-rpm contrib
     rpm ftp://ftp.riken.jp/Linux/scientific/ 304/i386/apt-rpm os updates contrib
  • SL 3.0.3 の場合 
     #apt-rpm pointing to SL
     #rpm ftp://ftp.scientificlinux.org//linux/scientific/ 303/i386/apt-rpm contrib
     rpm ftp://ftp.riken.jp/Linux/scientific/ 303/i386/apt-rpm os updates contrib

そして、

 apt-get update

を実行します。これで次回より、apt-get upgrade コマンドが使えるようになります。

SL 3.0.* から SL 3.0.x の最新版へアップグレードする方法は？

  • YUM
  1. rpm -Uvh ftp://ftp.scientificlinux.org/linux/scientific/30x/i386/misc/RPMS/yum-conf-30x.SL.noarch.rpm
  2. yum update yum
  3. yum upgrade
  • APT
    十分検証されていないため、YUMの利用を推奨します。 This section has not been tested yet.
  1. rpm -Uvh ftp://ftp.scientificlinux.org/linux/scientific/30x/i386/misc/RPMS/apt-sourceslist-30x.SL.i386.rpm
  2. apt-get update
  3. apt-get dist-upgrade
  4. apt-get upgrade-kernel

詳しくは、How To Upgrade To The Latest Release を参照してください。

SL 3.0.3 から SL 3.0.4 へアップグレードする方法は？

[YUM を使った手順]

 1. rpm -Uvh ftp://ftp.riken.jp/Linux/scientific/304/i386/SL/RPMS/yum-conf-304-2.SL.noarch.rpm
 2. yum update yum
 3. yum upgrade

[APT を使った手順]

 1. /etc/apt/sources.list.d/sl.list を開き、エディタで次のとおり書き換えます。
 
   #apt-rpm pointing to SL
   rpm ftp://ftp.riken.jp/Linux/scientific/ 304/i386/apt-rpm os updates contrib
   #rpm ftp://ftp.scientificlinux.org//linux/scientific/ 304/i386/apt-rpm contrib
 
 2. apt-get update
 3. apt-get dist-upgrade
  • [注意]
    APT を使ってのアップグレードは、十分検証されていない心配があります。
    もし apt-get dist-upgrade でエラーが出て中断する時は apt-get dist-upgrade --fix-missing を試してみること。

OpenOffice.org をアップグレードできません。

APT を使っての通常のアップグレード・コマンドでは、次のメッセージが出てアップグレードできない可能性があります。これは、openoffice.org- style-gnome (1.1.0-16.9.EL) も置き換える必要があるためです。

 # apt-get upgrade
 Reading Package Lists... Done
 Building Dependency Tree... Done
 The following packages have been kept back
    openoffice.org (1.1.0-16.14.EL => 1.1.2-24.2.0.EL3)
    openoffice.org-i18n (1.1.0-16.14.EL => 1.1.2-24.2.0.EL3)
    openoffice.org-libs (1.1.0-16.14.EL => 1.1.2-24.2.0.EL3)

この状況が発生した場合には、替わりに次のコマンドを使用すれば解決します。

 # apt-get dist-upgrade
