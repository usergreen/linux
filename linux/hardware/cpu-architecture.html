私の Linux マシンは 32ビット版か 64ビット版のどちらで稼働しているのですか？

グラフィカルなシステムモニター（あるいはビューア）を使用すれば、「width: 32 bits」などと表示されます。
あるいは、端末より次のコマンドを実行して、
$ uname -m
例えば「i686」と表示されたのなら、32ビットで稼働していることが確認できます。「X86_64」と表示されれば 64ビット版です。
↑

私のマシンは 64ビット版に対応していますか？

上記の項目と一見紛らわしいですが、これは使用中のパソコンの CPU 自体が 64ビットに対応しているのかという問題なので、全く別のはなしです。

現在（2013年2月時点で）発売されている最新のパソコンに搭載されている CPU は、ほぼ 64ビットに対応済みです。数年以上前の古いマシンでは、対応しているかどうかが問題になります。ふつう、メーカーのカタログや手元の取扱説明書の仕様表 を見れば、確認することができます。ウェブサイト上にも、対応一覧表があるので参考にしてください。
・PC Watch編集部 > Intelプロセッサー・ナンバー一覧表（http://pc.watch.impress.co.jp/docs/article/modelno/intelold.htm）

なお、64ビット対応であれば基本的に 32ビット用の OS も稼働させることができますが、その逆に 32ビットにしか対応していない CPU では 64ビット用の OSを稼働させることは不可能です（つまり逆の場合には互換性が有りません）。

PAE とは何のことですか？

物理アドレス拡張（Physical Address Extension）を指し、これは32ビットCPUで 4GB以上のメモリを扱う技術です。
従来、32ビット・マシンでは 4GB以上の大容量メモリを搭載することが事実上不可能でしたが、PAE を利用することで 64ビットマシン並に運用することができるようになりました。Linux の場合にも、Debian や RedHat をはじめとして、PAE をサポートしています。

詳しくは次の外部リンクを参照：
http://ja.wikipedia.org/wiki/%E7%89%A9%E7%90%86%E3%82%A2%E3%83%89%E3%83%AC%E3%82%B9%E6%8B%A1%E5%BC%B5

64ビット版へ乗り換えた方が良いのでしょうか？

それはお使いの環境に左右されます。後述の「武藤氏のページ」で書かれているように、一般的なユーザーであれば、現状で不便を感じていなければわざわざ乗 り換える 必要がないかも知れません。乗り換えた場合、特定のアプリケーションやデバイス・ドライバが機能しなくなり、システム自体が運用出来なくなるという可能性 を心配する必要があります。

したがって、いきなり重要なシステムへ導入を実施するのではなく、直ぐ元に戻しても支障がないようなマシンで前もってテストしておくことが大切です。
