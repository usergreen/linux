起動時、言語の自動判別 (auto-detect) に失敗します

Debian 10 で発生した事象です (2019-10-15)。プログラムの不具合で、言語の自動判別に失敗します。この結果、正しく言語設定されず、文字化けが発生して判読不能に陥りま す。デフォルトの自動判別の設定を止め、特定の言語を選択することで回避可能です。具体的には、最初のメニューの３つ目の ?? が設定 (Options) の項目なのでこれを選択し、次画面でメニュートップを選択、その後現れる一覧から Japanese を選択することで、すべて日本語に変換できます。

OpenTTD 1.8.0 の初期設定（例）について

パネルのメニューより　アプリケーション > ゲーム > OpenTTD　で起動。
基本設定という画面で以下のとおり項目を変更します。

自動車：左側通行
通貨単位：日本円
基本効果音セット：OpenSFX

設定が済んだら左上の x 印で設定画面を終了させ、初期画面へ戻ります。
「新しいゲーム」という項目を選択すると、地域設定の画面が表示されます。
「作成」というボタンを押すとゲームを開始します。

日本語フォントを変更するに は？

初期設定のフォントを変更する手順です。明朝体からゴシック体に直したい場合の参考にしてください。
openttd.cfg という名称のファイルは、OpenTTD の基本の設定ファイルです。このファイルは、Linux ディストリビューションでは以下の場所に存在します（参照：　http://wiki.openttd.org/Openttd.cfg ）。
 "~/.openttd"
もしくは以下の場所に在ります（Debian の場合はこちらにあります）。
 "~/.config/openttd/openttd.cfg"

日本語フォントを VL Pゴシックへ変更したいときは、18行目〜21行目を次のように書き換えます。ただし Debian 10 では、同じ手順で IPAゴシックや Note Sans CJK JP といったフォントへは変更できません（システム的な制約があるようです）。

 small_font = VL Pゴシック
 medium_font = VL Pゴシック
 large_font = VL Pゴシック
 mono_font = VL Pゴシック

Simutrans - 鉄道経営シミュレーション・ゲーム

これは、鉄道やバスなどの交通機関を運営するシミュレーション・ゲームのうちでは、広く人気があります。シミュレーションなので決まった ルールはありませんが、細かい操作の仕方を習得しないと充分楽しめません。以下の Tips を参照してください。

Simutrans の基本的なこと

・日本語解説サイト http://japanese.simutrans.com/ に丁寧で詳しい説明があります。まずはそちらを読んでください。

・simutrans-pak64 は基本のパックセットです。通常のインストールに含まれています。

・simutrans-pak128 はオプションで追加インストール可能です。pak128 は pak64 よりもグラフィックが細密なので画面表示がきれいです。

・代表的なパックセットのリストは、Simutrans 公式サイトの中の https://www.simutrans.com/en/paksets/ にあります。この中から選択すれば、不具合もあまり発生することなく楽しめます。この公式サイトの一覧にはありませんが、 pak.nippon という日本語版PAKセットもあって、この配布先は https://wa-st.github.io/pak-nippon/ です。

インストール方法 (Fedora 34)

Simutrans Germany universal repository が提供されています。これを yum のリポジトリに追加 すれば必要なパッケージを自動的に取得できます。参考資料: https://docs.fedoraproject.org/en-US/Fedora/17/html/System_Administrators_Guide/sec-Configuring_Yum_and_Yum_Repositories.html 　　　　　
         /etc/yum.repos.d/simutrans-germany-universal-repo

         [simutrans-germany.com]
         name=Simutrans-Germany universal Repo
         baseurl=https://simutrans-germany.com/simutrans/linux-package/rpm/
         enabled=1
         priority=1
         protect=0
         skip_if_unavailable=1
         gpgcheck=1
         repo_gpgcheck=1
         gpgkey=https://simutrans-germany.com/simutrans/linux-package/rpm/GPG-PUB-KEY.asc
上記のようにして /etc/yum.repos.d/ に追加します。pak128 も同時にインストールしたいのなら次のコマンドを実行します。
参照サイト: https://www.simutrans-forum.de/mybb/showthread.php?tid=8724 
         # dnf update
         # dnf install simutrans-uni-122 simutrans-uni-pak128

PAK セットをインストール（追加）する方法

・パックを個別に追加（インストール）したいときは、/usr/share/games/simutrans にあるフォルダの中へそのパックを追加（コピー）するだけで動作します。具体的には、次の例のようにコマンドラインからインストールできます。パックを フォルダごとインストールするには、-r オプションが必要です。また、必要に応じてユーザーがアクセス可能なように設定してください（元のファイルの構成や属性を一切変更しないときは -a オプション追加）。

 # cp -r -a ./pak.abc /usr/share/games/simutrans

・ユーザのゲームデータは、ユーザ・ディレクトリの ./.simutrans save/hoge.sve に保存されます。失っては困るような場合には、別の場所へバックアップしておくことをお推めします。

・pak128.britain を Debian 10 にインストールすると、次回以降ゲームが起動しなくなるという不具合を当方で確認しました（2019-10-14）。pak128 が正しく動作するために必要なパッケージが不足するというメッセージが初回起動時に出て、次回以降起動しなくなるというものです。そこで再インストールし ましたが、「パッケージが壊れています」という表示が出て治らなかったのですが、ユーザ・ディレクトリ ./.simutrans 以下の設定ファイルを全て削除すると、修復されて正常に起動するようになりました。ただし、この設定ファイルの中にある save/hoge.sve はユーザがゲームを継続するのに必要なデータなので、事前にデスクトップなどへバックアップしてください。pak128.britain だけを使う分には問題ないようです。

複数のパックセット (graphic set) をインストールしたとき

複数のパックセット (graphic set) をインストールした場合、ゲーム起動時に選択画面が表示されます。以前保存したゲームは、それぞれのパックセットの利用環境で利用しないと、互換性の問 題か ら動作障害を起こす可能性があります（必ず障害が発生するというわけではありません）。「以下のオブジェクト(pak)が不足しているため、転送 （障害をもたらす可能性があります：」というような警告が表示されたときは、いったんゲームを終了して、適切なパックセットを選び直して ください。これは、上記の pak128.britain を Debian 10 にインストールしたとき同様、気を付けてください。

＊＊＊＊＊ 具体例 ＊＊＊＊＊

pak64 で作成したゲームデータ → ◯ pak64 セットで起動
pak64 で作成したゲームデータ → ☓ pak128 セットで起動
pak128 で作成したゲームデータ → ◯ pak128 セットで起動
pak128 で作成したゲームデータ → ☓ pak64 セットで起動

起動時に PAKセットの選択画面を表示させない方法

　上記のように複数の PAK をインストールした場合、以下のような画面（例）が立ち上がります。
Choose one graphics set for playing:
 pak
 pak.nippon
To avoid seeing this dialogue define a path by:
 -adding 'pak_file_path = pak/' to your simuconf.tab
 -using '-objects pakxyz/' on the command line

デフォルトの設定を規定するファイルが /use/share/games/simutrans/config/simuconf.tab にあります。このファイルの37〜42行目には、起動時に選択されるグラフィック・セット（PAKセットのこと）とその設定例が記述されています。上に挙 げた例の場合であれば、行頭の # 記号を外し次のように書き換えれば、この選択画面なしで任意のパックセットを直接起動できるようになります。
#pak_file_path = pak/
#pak_file_path = pak.german/
#pak_file_path = pak128/
pak_file_path = pak.nippon/
#pak_file_path = pak.winter/
#pak_file_path = pak.ttd/

将棋ソフト「デーモン将棋」(Daemonshogi)

  • デーモン将棋のインストール
    Debian 3.1 (Sarge) / 4.0 (Etch) にインストールする場合は、DebianLinux-FAQ/Daemonshogi 参照。
  • メニューが文字化けして読めないときは？ (UTF-8 環境)
    UTF-8 に対応していないのが原因です。これに対処するには、EUC_JP の言語環境に変更して起動させてください。 
     $ LANG=ja_JP.EUC-JP daemonshogi
    英語のメニューで良いのなら、次のコマンドで起動させます。 
     $ LANG=C daemonshogi
    デスクトップメニュー（ランチャー）に登録するには、次のように env オプションを付ける必要があります。 
     env LANG=ja_JP.EUC-JP daemonshogi
    [重要] ja_JP.EUC_JP のロケールが利用可能でないと、メニューは日本語に出来ません。Debian Etch の場合 UTF-8 が標準のロケールであるため、# dpkg-reconfigure locales として EUC_JP の追加が必要です。
  • daemonshogi gtk2 version 
     daemonshogi-0.1.4gtk2-0.1.tar.gz
    http://www.nslabs.jp/daemonshogi-gtk2.rhtml にて公開されています。

XShogi (Linux向けで古くからある将棋ゲーム)

デスクトップに将棋盤が現れ、コンピュータと対局することができます。次の手順でインストールします（Debian 10 で検証済み）。

# apt install xshogi xfonts-100dpi

$ xshogi

XShogi の操作方法が判りにくいと思いますが下記コマンドで画面に説明が表示されます。オンライン・ヘルプもこちらで用意されています： https://www.gnu.org/software/gnushogi/manual/

$ xshogi -h
